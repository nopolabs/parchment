# Parchment API Key Auth Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Protect `POST /parchment/issue` with an `Authorization: Bearer` header checked against a per-environment Wrangler secret, and remove domain routes so parchment is reachable only via `workers.dev`.

**Architecture:** Add a two-line auth guard to `src/index.ts` immediately after the method check in the `/parchment/issue` handler. Remove the `[[env.mtw.routes]]` and `[[env.bbpp.routes]]` sections from `wrangler.toml`. Set `ISSUE_API_KEY` as a Wrangler secret per environment and deploy. The client Pages Functions (mtw4, bbpp) will add the header; direct `workers.dev` callers without the key get 401.

**Tech Stack:** TypeScript strict mode, Cloudflare Workers, Wrangler 3.x

**Deploy this plan before the mtw4 and bbpp plans.** After deploying, the domain endpoints (mastertimewaster.com/parchment/*, bigbeautifulpeaceprize.com/parchment/*) will 404 until the client Pages Functions are deployed. Keep the window short.

### Task 1: Add auth guard to `src/index.ts`

**Files:**
- Modify: `src/index.ts`

- [ ] **Step 1: Locate the issue handler**

Open `src/index.ts`. The `/parchment/issue` handler starts around line 77:

```typescript
    if (url.pathname === '/parchment/issue') {
      if (method !== 'POST') return jsonError(405, { error: 'method not allowed' });

      let name: string;
```

- [ ] **Step 2: Insert the auth guard**

Add the auth check between the method check and `let name: string;`. The `Env` type is generated by `wrangler types` and does not include secrets — follow the same cast pattern already used in `src/queue.ts` for `RESEND_API_KEY`:

```typescript
    if (url.pathname === '/parchment/issue') {
      if (method !== 'POST') return jsonError(405, { error: 'method not allowed' });

      const apiKey     = (env as Env & { ISSUE_API_KEY?: string }).ISSUE_API_KEY;
      const authHeader = request.headers.get('Authorization');
      if (!authHeader || !apiKey || authHeader !== `Bearer ${apiKey}`) {
        return jsonError(401, { error: 'unauthorized' });
      }

      let name: string;
```

- [ ] **Step 3: Run typecheck and lint**

```bash
npm run typecheck
npm run lint
```

Expected: no errors from either command.

- [ ] **Step 4: Commit**

```bash
git add src/index.ts
git commit -m "feat: add ISSUE_API_KEY auth guard to POST /parchment/issue"
```

### Task 3: Generate keys, set secrets, and deploy

- [ ] **Step 1: Generate two independent API keys**

```bash
echo "MTW key:  $(openssl rand -hex 32)"
echo "BBPP key: $(openssl rand -hex 32)"
```

Save both values securely (e.g., 1Password). You will need the MTW key when setting up the mtw4 Pages secrets, and the BBPP key for the bbpp Pages secrets.

- [ ] **Step 2: Set the Wrangler secret for each environment**

```bash
echo '<paste-mtw-key-here>'  | npx wrangler secret put ISSUE_API_KEY --env mtw
echo '<paste-bbpp-key-here>' | npx wrangler secret put ISSUE_API_KEY --env bbpp
```

Each command will prompt to confirm. Expected output ends with: `✨ Success! Uploaded secret ISSUE_API_KEY`

- [ ] **Step 3: Deploy both environments**

```bash
npm run deploy:mtw
npm run deploy:bbpp
```

Expected: each ends with `Deployed parchment-worker-mtw` / `parchment-worker-bbpp`.

- [ ] **Step 4: Verify the auth guard is live on workers.dev**

Find your workers.dev subdomain in the Cloudflare dashboard (Workers & Pages → the worker → the workers.dev URL). Then:

```bash
# Should return 401 — no key
curl -s -X POST "https://parchment-worker-mtw.<your-account>.workers.dev/parchment/issue" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "name=Test&email=test@example.com"
# Expected: {"error":"unauthorized"}

# Should return 202 — correct key
curl -s -X POST "https://parchment-worker-mtw.<your-account>.workers.dev/parchment/issue" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -H "Authorization: Bearer <paste-mtw-key-here>" \
  -d "name=Test User&email=test@example.com"
# Expected: {"status":"queued"}
```

- [ ] **Step 5: Verify health and render still work without auth**

```bash
curl -s "https://parchment-worker-mtw.<your-account>.workers.dev/parchment/health"
# Expected: {"status":"ok","siteId":"mtw"}

curl -s "https://parchment-worker-mtw.<your-account>.workers.dev/parchment/render?name=Test" \
  --output /tmp/test.png && file /tmp/test.png
# Expected: PNG image data
```
